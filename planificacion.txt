================================================================================
 1. DESCRIPCIÓN GENERAL
================================================================================

  Nombre del sistema:   SistemaOF
  Versión:              3.0
  Tecnología:           PHP (MVC) + MySQL/MariaDB
  Servidor:             Apache/Nginx — cPanel compatible
  Versión PHP:          8.4+
  Versión MariaDB:      11.4+
  Charset:              utf8mb4 (nuevas tablas) / latin1 (tablas originales)

  Empresas operadoras:
  ├── SERVICIOS DE CONSTRUCCION O&F S. DE R.L.
  └── PINTER S. DE R.L.

  Propósito:
  Sistema administrativo modular para la gestión integral de proyectos de
  construcción, remodelación y servicios. Controla presupuestos, alcances,
  cotizaciones, compras, entregas, planillas de pago y reportes financieros,
  con seguridad basada en roles y permisos granulares (RBAC).

 4. SEGURIDAD — MÓDULO RBAC
================================================================================

  MODELO DE SEGURIDAD:
  ├── sof_users           Usuarios del sistema
  ├── sof_roles           Roles disponibles
  ├── sof_modulos         Módulos del sistema
  ├── sof_acciones        Acciones por módulo
  ├── sof_user_roles      Asignación usuario→rol (N:N)
  ├── sof_role_permissions Permisos rol→módulo→acción
  ├── sof_audit_logs      Bitácora de acciones
  ├── sof_user_sessions   Sesiones activas
  └── sof_password_resets Recuperación de contraseña

  REGLAS DE ACCESO:
  ├── Un usuario puede tener múltiples roles
  ├── Un rol puede tener múltiples módulos
  ├── Cada módulo puede tener múltiples acciones
  ├── Los permisos afectan: menús, rutas, botones y operaciones
  └── El Superadmin tiene acceso total sin restricciones

  ROLES INICIALES (8):
  ├── Superadmin      → Control total, es_superadmin = 1
  ├── Gerencia        → Acceso gerencial y aprobaciones
  ├── Administrador   → Administración operativa completa
  ├── Compras         → Gestión de compras y órdenes
  ├── Ingeniería      → Gestión de proyectos y alcances
  ├── Finanzas        → Control financiero y planillas
  ├── Cotizaciones    → Creación y gestión de cotizaciones
  └── Consulta        → Solo lectura

  ACCIONES DISPONIBLES (10):
  VER · CREAR · EDITAR · ELIMINAR · APROBAR
  IMPRIMIR · EXPORTAR · ANULAR · CERRAR · REABRIR

  PANTALLAS DE SEGURIDAD:
  ├── Login con hash de contraseña (bcrypt/argon2)
  ├── Recuperación de contraseña por token
  ├── Cambio de clave
  ├── Gestión de usuarios (CRUD)
  ├── Gestión de roles (CRUD)
  ├── Asignación de módulos a roles
  ├── Asignación de acciones por módulo/rol
  ├── Bitácora de acciones (filtrable)
  ├── Sesiones activas (ver y cerrar)
  └── Perfil de usuario

 8. FLUJO COMPLETO DEL NEGOCIO
================================================================================

  PASO 1: Registro de cliente
  └── Módulo Clientes → crear cliente con RTN y datos de contacto

  PASO 2: Crear proyecto
  └── Módulo Proyectos → asignar cliente, empresa, ingeniero(s),
      fechas, código y dirección

  PASO 3: Definir alcances y costos
  └── Módulo Alcances → por línea de servicio, definir:
      materiales, mano de obra, costos indirectos,
      % de utilidad y precio de oferta

  PASO 4: Generar cotización
  └── Módulo Cotizaciones → crear documento comercial
      a partir del alcance, con ítems, descuentos e ISV

  PASO 5: Aprobación del cliente
  └── Cotización enviada → marcar como facturada al aprobar

  PASO 6: Emisión de órdenes de compra
  └── Módulo Compras → OC por proveedor con ítems,
      forma de pago, ingeniero solicitante,
